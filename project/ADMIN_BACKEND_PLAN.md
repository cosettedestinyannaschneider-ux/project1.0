# 管理员后端与数据库实施计划

## 1. 当前状态

- 管理员前端 UI 已完成，当前业务数据主要来自 `uni-app-frontend/common/admin-mock-data.js`。
- 管理员前端包含：工作台、用户与组织管理、知识库、AI 模型配置、企业档案、操作日志、报告模板、数据备份。
- 后端已有部分接口，但数据库组织关系、权限持久化、企业综合档案、AI Key 安全和备份模块仍需改造。
- 本阶段目标是完成管理员后端、数据库迁移和管理员前后端真实接口联调，再开始普通用户 UI 改造。

## 2. 实施原则

1. 严格使用现有 Node.js + Express + MySQL 分层结构。
2. 路由放在 `backend/server.js`，业务逻辑放在 `backend/bll/`，数据库访问放在 `backend/dal/`。
3. 所有代码、SQL、接口和复杂逻辑必须包含清晰中文注释。
4. 前后端严格分离，后端不得依赖前端模拟数据。
5. 每个后端阶段必须同步更新：
   - `API_DOC.md`
   - `DESIGN_DOC.md`
   - `BACKEND_TODO.md`
   - `TEST_RECORD.md`
   - `database/schema.sql`
6. 每个阶段完成后先联调对应管理员页面，不一次性修改全部后端。
7. 禁止使用向量数据库、RAG、LangChain、消息队列和多租户架构。

## 3. 推荐执行顺序

### 后端阶段 A：数据库迁移与组织关系

**优先级：最高**

#### 数据库任务

- 将企业调整为组织主数据，不再以 `enterprises.user_id` 作为企业归属依据。
- `departments` 增加 `enterprise_id`，建立企业 1:N 部门关系。
- 用户继续通过 `users.department_id` 关联部门，并由部门推导所属企业。
- 设计并实现用户权限持久化。
  - 推荐新建 `user_permissions` 关联表。
  - 权限 Key：`enterprise:manage`、`image:manage`、`analysis:run`、`report:download`、`knowledge:view`。
- `enterprises` 增加 `inspection_status`。
- `ai_model_configs` 增加 `provider`。
- 新增 `backup_records` 表。
- 编写可重复执行的数据迁移脚本，并在修改前备份现有数据。

#### 验收条件

- 企业、部门和用户关系满足：企业 1:N 部门，部门 1:N 用户。
- 旧数据迁移后无孤立用户、孤立部门和重复企业。
- `database/schema.sql` 可完整创建最新数据库结构。
- 数据库变更说明与迁移测试记录完整。

### 后端阶段 B：用户、企业、部门与权限管理

**优先级：最高**

#### 接口范围

- 用户：列表、新增、更新、删除。
- 企业组织：列表、新增、重命名、删除。
- 部门：列表、新增、更新、删除。
- 权限：随用户新增和更新保存，用户列表返回权限集合。

#### 关键规则

- 企业存在部门、用户或业务档案时禁止删除。
- 部门存在用户时禁止删除。
- 后端拒绝用户关联其他企业的部门。
- 禁止删除当前登录管理员。
- 所有修改和删除操作写入 `action_logs`。

#### 联调页面

- `uni-app-frontend/pages/admin/users.vue`

#### 验收条件

- 管理员可完成企业创建、部门创建、用户创建与权限分配完整流程。
- H5 和微信小程序均使用真实接口完成组织管理。

### 后端阶段 C：企业详细档案与专属排查档案

**优先级：高**

#### 接口范围

- 企业完整档案更新：`POST /api/admin/enterprises/update`
- 企业综合查询：`POST /api/admin/enterprises/query`
- 企业查询结果导出：`POST /api/admin/enterprises/export`

#### 数据关联规则

- 隐患图片通过 `hazard_images.enterprise_id` 关联企业。
- AI 分析结果和报告通过 `inspection_reports.enterprise_id` 关联企业。
- 报告图片通过 `inspection_report_images` 关联。
- 上传图片、执行分析和生成报告时必须校验并写入 `enterprise_id`。
- 企业查询接口聚合图片数、分析数、报告数、隐患整改统计和主要隐患类型。

#### 权限规则

- 管理员可查看和修改全部企业。
- 有 `enterprise:manage` 权限的普通用户仅可修改所属企业详细档案。
- 企业名称、组织关系等关键主数据建议仅管理员修改。

#### 联调页面

- `uni-app-frontend/pages/admin/enterprises.vue`

#### 验收条件

- 企业信息修改后返回并显示最新数据。
- 可按排查状态和所属区域分类查询。
- 企业专属档案可查询关联图片、分析结果与 Word/PDF 报告。

### 后端阶段 D：知识库与 AI 模型配置

**优先级：高**

#### 知识库任务

- 联调知识条目和分类 CRUD。
- 新增批量归档接口。
- 分类删除前校验关联文档。
- 文件上传校验扩展名、MIME 类型和大小。

#### AI 配置安全整改

- API Key 必须真正加密存储。
- 列表和当前配置接口禁止返回原始密钥字段，仅返回脱敏值。
- 更新接口使用允许字段白名单。
- 后端禁止删除当前启用模型。
- 激活模型使用事务，保证仅一个配置启用。

#### 联调页面

- `uni-app-frontend/pages/admin/knowledge.vue`
- `uni-app-frontend/pages/admin/model-config.vue`

### 后端阶段 E：日志、报告模板与数据备份

**优先级：中**

#### 操作日志

- 支持关键词、模块、时间范围和分页筛选。
- 所有管理员关键操作写入日志。

#### 报告模板

- 使用 Multipart 上传 DOCX。
- 校验文件类型并安全保存。
- 禁止删除默认模板。
- 设置默认模板使用事务。

#### 数据备份

- 实现手动备份、备份记录、自动备份策略接口。
- 自动备份使用后端定时任务，不使用消息队列。
- 本阶段不提供数据库恢复接口。

#### 联调页面

- `uni-app-frontend/pages/admin/logs.vue`
- `uni-app-frontend/pages/admin/templates.vue`
- `uni-app-frontend/pages/admin/backup.vue`

### 后端阶段 F：管理员工作台统计与最终联调

**优先级：中**

- 提供企业总数、待分析数量、报告总数等管理员统计接口。
- 替换管理员工作台模拟数据。
- 执行管理员全流程 H5 与微信小程序联调。
- 完成功能测试、安全测试和数据库迁移测试记录。

## 7. 风险提示

- `enterprises.user_id` 是历史兼容字段，阶段 B 禁止继续写入或作为企业归属依据。
- 用户所属企业必须通过部门关系推导，禁止在用户表重复保存企业 ID。
- 权限控制必须由后端强制执行，不能只依赖前端按钮隐藏。
- 用户、权限和组织关系的多表修改必须使用事务。
- API Key 当前存在明文存储和返回风险，进入 AI 配置阶段时必须优先整改。
- 企业和部门删除必须先验证所有关联数据，禁止依赖数据库异常作为业务提示。
