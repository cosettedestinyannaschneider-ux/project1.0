# AGENTS.md - 安全生产社会化服务智检系统开发规则

## 一、项目概况

- **项目名称**：安全生产社会化服务智检系统
- **第一阶段**：微信小程序 + H5 Web 双端应用，企业详情为核心入口
- **核心功能**：企业信息、隐患图片、AI 识图分析、知识库检索、分析结果、报告生成
- **AI 模型**：豆包视觉模型（后续扩展 DeepSeek、阿里千问）
- **知识库检索**：MySQL 关键词匹配，第一阶段不使用向量数据库、RAG 或消息队列
- **技术栈**：
  - 前端：uni-app + Vue 3 + Vite
  - 后端：Node.js + Express，端口 3000
  - 数据库：MySQL 8.0，库 `ai_project`
  - 认证：scrypt 密码哈希 + 会话持久化（sessions 表）
  - 报告生成：docx 库 + docx-templater，PDF 用 pdfkit

## 二、通用规则

1. 所有代码必须使用中文注释，包括类、方法、接口、逻辑和 SQL
2. 代码结构规范，可维护、可扩展、低耦合高内聚
3. 前后端分离，目录结构清晰
4. 每个 Phase 完成后，输出该 Phase 对应的接口文档、数据库变更说明、功能测试记录
5. 每个 Phase 完成后，确保 H5 端完整流程可跑通，微信小程序端同步适配

## 四、文档规范

### 开发文档
- 模块功能说明、业务流程、使用方法
- 每个 Phase 完成后更新

### 接口文档
- 接口名称、路径、请求方式、参数、返回值、示例
- 存放于 `project/API_DOC.md`

### 数据库设计文档
- 表结构、字段说明、索引、表关系
- 存放于 `project/DESIGN_DOC.md`

### 界面设计文档
- 页面名称、功能、字段、交互说明

### 文档格式
- 用户报告（隐患排查报告）：Word (.docx) + PDF (.pdf) 双格式，Word 可编辑，PDF 存档
- 开发文档（接口文档、数据库设计、测试记录）：Markdown 存放于项目仓库

## 五、测试规范

- **功能测试**：用例、预期结果、实际结果、是否通过
- **性能测试**：响应时间、并发情况、大数据量测试（核心功能操作响应时间控制在合理范围）
- **安全测试**：密码校验、会话验证、权限拦截、数据隔离、文件上传类型校验、SQL 注入防护、XSS 防护
- **问题整改**：问题描述、原因分析、解决方案、整改状态

## 六、异常处理

- **知识库解析异常**：记录解析状态和错误原因，提示管理员重新上传或手动处理
- **AI 识图失败**：记录任务失败原因，前端提示网络异常/模型超时/配置错误，提供重试入口
- **知识库无匹配**：提示"未找到相关法规条款"，仍生成隐患描述和整改建议，依据字段标记为空或待补充
- **报告生成异常**：模板缺失、占位符错误或 docx-templater 生成失败，记录日志并提示用户检查模板或稍后重试

## 七、第一阶段限制

- 不使用向量数据库、RAG、LangChain
- 不使用消息队列、多租户、部门级数据隔离
- 不处理云服务器部署、HTTPS、微信合法域名配置
- `confidence` 字段记录 AI 置信度，但不作为业务判断依据
- 数据隔离：管理员查看全部数据，普通用户仅查看 `create_user_id = 自身` 的数据
- 知识库检索仅使用 MySQL LIKE / 全文索引关键词匹配

## 八、输出要求

- 每次生成代码必须包含中文注释
- 保证代码结构规范，低耦合高内聚
- 每个 Phase 同步输出：接口文档变更、数据库变更说明（DDL 差异）、功能测试记录
- 用户隐患排查报告：同时输出 Word (.docx) 和 PDF (.pdf)
  - Word：`/uploads/reports/word/` — 可编辑版本
  - PDF：`/uploads/reports/pdf/` — 存档版本
- 前端下载提供 Word / PDF 双按钮
